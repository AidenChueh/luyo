第 1 部 — 產品定位與設計系統

1.1 產品定位

  • 一站式個人旅遊工具：涵蓋「規劃 → 執行 → 記錄 → 回顧」完整生命週期
  • 核心使用情境：行前籌備、旅途中即時記帳與看行程、行後生成總結報告
  • 單人為主、支援同行者協作（分帳、共同編輯）

1.2 設計原則（Apple HIG 為骨幹）

  • Clarity 清晰：內容優先，文字易讀、圖示語意明確、留白充足
  • Deference 謙遜：UI 不搶戲，照片與內容是主角；柔和陰影而非重邊框
  • Depth 層次：用層級、轉場與動態建立空間感（卡片浮起、頁面滑入）
  • 直覺導航：底部 5 分頁 + 頁內快速入口，任何功能 ≤ 2 次點擊可達
  • 無障礙：見 1.7

1.3 視覺風格混搭

來源借鏡之處
Airbnb大圖卡片、圓角、柔和陰影、溫暖親近的攝影感、清楚的價格/資訊層級
Google Travel結構化行程時間軸、地圖整合、資訊密度與分類標籤的處理
Notion乾淨的排版、區塊化內容、Markdown 編輯、低調的灰階介面框架
原則：放鬆、乾淨、專業的旅遊感。明確排除過度漸層光暈、紫藍科技風、塞滿的霓虹色塊、無意義的 3D 與「AI 感」裝飾。照片用真實旅遊攝影風，不用合成插畫味重的素材。

1.5 形狀、陰影、間距、字體

  • 圓角：卡片 / 圖片 / 大容器 16px；按鈕 / 輸入框 12px；標籤 / 小元件 8px；頭像圓形
  • 陰影：柔和多層，低不透明度（例 0 4px 16px rgba(28,25,23,0.08)）；不用硬黑邊；浮起卡片陰影加深
  • 間距：8pt 網格（4 / 8 / 12 / 16 / 24 / 32）
  • 字體：系統字（-apple-system / SF / Roboto）+ Noto Sans TC；標題粗、內文常規；數字（金額）可用等寬對齊
  • 圖示：單一線性圖示集（如 SF Symbols / Lucide），線寬一致

1.6 共用元件庫（Design System）

  • 按鈕：主要 / 次要 / 文字 / 圖示 / 浮動操作鈕（FAB，用於「新增」）
  • 卡片：旅程卡、地點卡、行程項目卡、花費卡、機票卡、住宿卡
  • 標籤 Chip / Badge：分類、狀態、標籤（must-go / 替代 / 已完成）
  • 進度條 / 環形圖：預算用量
  • 輸入元件：文字、金額（含幣別選擇）、日期 / 時間、評分（星等）、心情、照片上傳
  • 清單列：可勾選、可拖曳、左右滑出操作（編輯 / 刪除 / 收藏）
  • 底部彈出 Sheet、Modal、Toast、空狀態、載入骨架屏
  • 時間軸元件（行程、日誌共用）

1.7 無障礙與響應式

  • 對比度：文字 ≥ WCAG AA（4.5:1），大字 ≥ 3:1
  • 觸控目標：≥ 44×44pt
  • 動態字體：支援系統字級縮放，版面不破
  • 語意標籤：VoiceOver / TalkBack 朗讀標籤、圖示有替代文字
  • 不只靠顏色傳達狀態：同時用圖示 / 文字（色盲友善）
  • 深色模式：完整對應（見進階功能），暖色調在深色下降飽和
  • Mobile-first 斷點：手機（單欄）→ 平板（雙欄）→ 桌面（側欄 + 主區）；底部導航在大螢幕可轉為側邊導航

第 3 部 — 功能模組

每個模組格式：目的 → 版面結構 → 欄位 / 資料 → 互動 / 邏輯 → 設計註記。

3.1 首頁儀表板 Home Dashboard

目的

一眼掌握所有旅程狀態，快速進入或建立旅程。

版面結構

  • 頂部：問候 / 搜尋列
  • 進行中 / 即將出發旅程：大封面卡（含倒數）置頂
  • 其餘旅程：卡片列表 / 網格
  • 右下 FAB：新增旅程

每張旅程卡欄位

  • 封面照片、旅程名稱、國家 / 城市
  • 日期區間、總天數、出發倒數（D-day）
  • 預算 / 已花 / 結餘（小型進度條）
  • 狀態徽章：規劃中 / 進行中 / 已完成
  • 收藏星標

互動 / 邏輯

  • 搜尋：依名稱 / 國家 / 城市
  • 排序：出發日 / 建立日 / 名稱 / 預算
  • 篩選：狀態、收藏
  • 收藏：星標置頂
  • 封存：完成的旅程歸入「封存」不佔主列表，可還原
  • 狀態自動推斷：依今日與日期區間自動切換規劃中 / 進行中 / 已完成（可手動覆寫）
設計註記：進行中旅程用主色描邊 + 倒數醒目；已完成旅程降低彩度。空狀態引導「建立第一趟旅程」並可選 AI 規劃。

3.2 旅程總覽 Trip Overview

目的

單一旅程的中央樞紐，總覽 + 通往所有子功能。

版面結構

  • Hero 大圖橫幅：封面 + 目的地 + 日期區間 + 天數 + 出發倒數
  • 天氣卡：目的地天氣（當前 / 預報）
  • 預算進度條 + 花費統計（總額 / 已花 / 結餘 / 百分比）
  • 關鍵數據：已完成行程數、收藏地點數、相片數
  • 6 個快速入口：行程、地圖、記帳、日誌、清單、地點

互動 / 邏輯

  • 快速入口點擊進入對應模組（L3）
  • 倒數結束（進行中）改顯示「Day N / 共 M 天」
  • 天氣串接外部 API（待選供應商）
設計註記：此頁是「看板」不是「操作台」——資訊卡為主、動作集中在快速入口。

3.3 行程規劃 Itinerary Planning

目的

逐日時間軸排程，是 App 的核心。

版面結構

  • 頂部：日期分頁 / 橫向日曆（Day 1、Day 2…）
  • 主體：當日垂直時間軸，依時間排列的行程項目卡
  • 項目間自動顯示「空檔 X 小時」與「移動時間」

單一行程項目欄位

  • 開始 / 結束時間
  • 標題、地點（連結地點庫）
  • Google Maps 連結
  • 分類：景點 / 餐廳 / 交通 / 購物 / 住宿（含分類色與圖示）
  • 備註、預估花費 / 實際花費
  • 照片、評分（星等）

互動 / 邏輯

  • 拖曳排序、調整時間
  • 複製項目到其他日期
  • 自動計算空檔時間
  • 時間衝突警示（同時段重疊 → 磚紅提示）
  • 項目與記帳連動：填實際花費可一鍵建立花費紀錄
  • 項目與地點庫連動：選地點自動帶入名稱 / 地圖 / 分類
設計註記：時間軸左側為時間刻度，右側卡片；分類色作為卡片左緣色條。衝突與空檔用非紅色以外的輔助圖示，避免只靠顏色。

3.4 地圖檢視 Map View

目的

空間視角看地點與每日動線。

內容 / 標記類型

  • 景點、餐廳、飯店、車站、機場（不同圖示 / 顏色標記）

功能

  • 地圖標記（依分類）
  • 路線規劃、移動時間、距離計算
  • 每日動線視覺化（依 Day 篩選，連線顯示順序）
  • 點標記 → 彈出地點卡（可跳行程 / 地點庫）
設計註記：地圖供應商（Google Maps / Mapbox）待選，影響成本與離線能力。每日動線用該日分類色的連線。

3.5 記帳 Expense Tracker

目的

旅途即時記帳 + 花費分析。

分類

機票、住宿、交通、餐飲、門票、購物、伴手禮、保險、網卡 / SIM、其他（各有圖示與色）。

單筆欄位

  • 日期、金額（多幣別）、分類、備註、照片（收據）、地點
  • 付款人 / 分攤對象（連同行者，供分帳）

統計

  • 總額、每日花費、各分類佔比、每人平均、剩餘預算

圖表

  • 圓餅圖（分類佔比）、長條圖（每日 / 分類）、趨勢圖（累積 / 折線）

互動 / 邏輯

  • 多幣別：每筆記原始幣別，依匯率換算主幣別彙總
  • 快速記一筆（底部 Sheet，金額 + 分類兩步完成）
  • 與行程項目、預算（3.6）雙向連動
設計註記：記帳是高頻動作，入口要極短；金額輸入用大數字鍵盤，幣別預設旅程主幣別。

3.6 預算管理 Budget Management

目的

設定與監控總 / 每日預算，及早預警超支。

功能

  • 設定總預算、每日預算
  • 顯示用量百分比、剩餘、超支金額
  • 分級警示：50% / 75% / 90% / 超支（對應琥珀 → 磚紅 + 推播）
  • 分類預算（選配）：各分類設上限
設計註記：與記帳同源資料；警示用環形進度 + 文字 + 圖示三重表達。可接 AI 預算分析（3.14）。

3.7 旅遊日誌 Travel Journal

目的

逐日記錄心情與見聞。

單篇欄位

  • 標題、內文（Markdown）、心情評分、照片 / 影片、日期

互動 / 邏輯

  • Markdown 編輯 + 即時預覽
  • 自動儲存（草稿）
  • 時間軸檢視（依日期串起）
設計註記：編輯體驗對齊 Notion——乾淨、區塊化、低干擾。心情評分用表情 / 圖示等級。

3.8 相簿 Photo Gallery

目的

集中瀏覽旅程照片。

功能

  • 依日分組
  • 瀑布流（masonry）版面
  • 大圖預覽、說明文字（caption）、地點標籤
設計註記：照片是主角，介面退到背景（深色預覽底）。可從行程 / 日誌的照片聚合而來。

3.9 行前準備中心 Preparation Center

3.9.1 待辦清單 To-do

  • 項目（簽證、保險、換匯、餐廳訂位…）、優先級、截止日、狀態

3.9.2 打包清單 Packing

  • 分類：證件、衣物、電子、盥洗、藥品；可勾選（checkbox）

3.9.3 購物清單 Shopping

  • 品名、數量、預估價格、是否已購
設計註記：三種清單共用「可勾選清單列」元件；待辦可接近截止日推播提醒。提供預設範本（首次建立旅程帶入常見項目）。

3.10 機票管理 Flight Manager

欄位（去程 / 回程卡片）

  • 航空公司、航班編號
  • 起飛 / 抵達時間、航廈 Terminal、座位、行李額度
設計註記：登機證式卡片版型（Airbnb 卡片感 + 票券分隔線）。去程 / 回程並列或分頁。可連動行程時間軸的交通項目。

3.11 住宿 Accommodation

欄位

  • 名稱、地址、電話、訂房編號
  • 入 / 退房時間、價格、官網連結、地圖位置
  • 房間照片、已收藏住宿
設計註記：地圖位置與 3.4 地圖共用；可一鍵把入退房加入行程時間軸。

3.12 地點庫 Places Library

欄位

  • 類型：景點、餐廳、咖啡廳、商場
  • 標籤：必去 must-go / 替代 alternative / 已完成 completed
  • 照片、評分、備註、Google Maps 連結

互動

  • 收藏的地點可直接加入行程（3.3）與地圖（3.4）
  • 依類型 / 標籤篩選
設計註記：地點庫是行程與地圖的「素材庫」，一處維護、多處複用。

3.13 同行者 Companions

欄位 / 功能

  • 新增同行者：姓名、頭像、聯絡方式
  • 分帳：花費可指定付款人與分攤對象
  • 債務追蹤：誰欠誰多少，結算建議（最少筆數還清）
設計註記：與記帳（3.5）緊密耦合；提供「誰付 / 大家平分 / 自訂比例」三種分攤模式。

3.14 旅程總結 Trip Summary（行後自動生成）

內容

  • 總天數、總花費、每日平均花費
  • 造訪地點數、餐廳數、照片數、最愛地點
  • 視覺化報告卡（社群分享格式，例如 9:16 / 1:1）

互動

  • 旅程完成後自動產生
  • 生成可分享圖卡 → 匯出 / 分享
設計註記：報告卡走「精選回顧」風（大數字 + 代表照片 + 暖色版型），不堆砌資訊。資料全部來自既有模組彙總。

3.15 AI 功能

3.15.1 AI 行程規劃

  • 輸入：目的地、天數、預算（及偏好 / 同行人數）
  • 輸出：行程建議、預算分配、每日排程草稿（可一鍵套入 3.3）

3.15.2 AI 預算分析

  • 找出高花費分類、預算風險、超支警示與建議

3.15.3 AI 旅遊回顧

  • 生成每日亮點、花費摘要、旅遊故事（敘事文，可放入日誌 / 總結）
設計註記：AI 為「輔助」而非主導——所有 AI 產出皆為可編輯草稿，使用者保留最終決定。模型供應商與成本待定（建議用最新 Claude 模型）。需處理離線 / 失敗的降級。

Photo 照片檔案、說明、地點、日期屬 Trip；來源可為行程 / 日誌
Place 地點類型、標籤、評分、備註、Maps、座標屬 Trip（或全域收藏）
Flight 機票航司、航班、起訖、航廈、座位、行李屬 Trip
Accommodation 住宿名稱、地址、訂房號、入退房、價格、座標屬 Trip
ChecklistItem 清單類型(待辦/打包/購物)、內容、優先 / 截止 / 數量 / 價格、狀態屬 Trip
Companion 同行者姓名、頭像、聯絡屬 Trip；連 Expense 分帳
Currency / Rate 匯率幣別、匯率、時間供換算與歷史

第 5 部 — 進階與技術功能

  • 離線模式：本地優先，無網路可看 / 可記，恢復連線再同步
  • 多幣別換算 + 匯率歷史：每筆記原幣別，保存當下匯率，提供歷史查詢
  • 匯出：PDF / Excel（行程、花費、總結）
  • 旅程分享 + 協作編輯：產生分享連結，多人共編（與同行者）
  • 深色模式：完整對應，暖色於深色降飽和
  • PWA 支援：可安裝、離線快取
  • 雲端同步：跨裝置；需處理衝突合併
  • 推播通知：預算警示、待辦截止、出發倒數
  • 倒數小工具 Widget：桌面 / 鎖屏顯示出發倒數
技術註記（待決策）：前端框架、地圖供應商、後端 / 同步方案（自架 vs BaaS）、AI 供應商皆未定。離線 + 雲端同步是架構成本最高的一塊，建議早期就決定「本地優先」資料層。
